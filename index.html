The framework is organized into 10 key categories of the ontology lifecycle (e.g., Availability, Documentation, Maintenance). Each category includes one Principle, which may have one or more Requirements, and each requirement is supported by one or more Guidelines.

‚úÖ Step 3 ‚Äì Select the Governance Elements

Use the dropdown sections to explore each category. You can select:

  • Principles ‚Äì Foundational ideas that guide the decision-making process throughout the ontology lifecycle.
  • Requirements ‚Äì Conditions and statements that express a need and must be fulfilled to comply with the principles.
  • Guidelines ‚Äì Group of recommendations and methodological suggestions to support the development of the resource based on the principles and requirements established in the first place.

WARNING! A Guideline can only be selected if its Requirement and Principle are also selected. Similarly, a Requirement depends on its principle. If for example, a Requirement is selected without selecting its Principle, then it won't be included in the file exported. This ensures logical consistency in your governance model.
